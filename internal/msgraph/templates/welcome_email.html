GitHub Security Best Practices

  • Enable Two-Factor Authentication: 2FA is required for all members. Please ensure it's set up on your GitHub account.
  • Use SSH keys: For repository access, use SSH keys instead of passwords whenever possible.
  • Be careful with secrets: Never commit API keys, passwords, or other secrets to repositories.
  • Keep dependencies updated: Regularly update dependencies to avoid security vulnerabilities.
  • Review code changes: All code should be peer-reviewed before merging to main branches.
  • Understand repository permissions: Only request access to repositories you need to work with.
